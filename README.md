Una empresa comercial desea desarrollar un sistema para registrar información de sus
vendedores y analizar el resultado de una venta realizada.
De cada vendedor interesa conocer su nombre, su área de trabajo, el monto total vendido, el porcentaje de comisión y el cumplimiento de meta.

La empresa necesita que cierta información general pueda visualizarse fácilmente, pero los datos económicos deben mantenerse protegidos para evitar accesos directos indebidos.
Además, el sistema debe permitir calcular:

el valor de la comisión del vendedor,
el ingreso total que recibe sumando comisión,
el estado de cumplimiento según el porcentaje alcanzado,
y un mensaje final de desempeño.

La empresa también desea que la venta registrada pueda generar una salida clara en pantalla, mostrando los datos del vendedor, los resultados del cálculo y una interpretación del desempeño alcanzado.
El diseño debe considerar una estructura orientada a objetos, donde exista separación adecuada de responsabilidades entre las clases y se evidencie el uso correcto de encapsulamiento.
